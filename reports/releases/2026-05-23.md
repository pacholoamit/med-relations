## Planned Releases

### Add sync modes across data-factory v1 + v2 UIs with adapter-managed defaults

Added per-dataset Sync Mode (Full Refresh / Incremental) and Write Mode (Append, Overwrite, Append+Dedup, Overwrite+Dedup) configuration to both the Data Factory v1 and v2 UIs. Adapter-managed defaults handle primary keys, cursor columns, and dedup strategy automatically so users only make the two decisions that matter. Also added a column-mapping warning before ingestion fires on unmapped datasets, and ClickHouse data wipe on dataset deletion.

### Campaign Requestor Enhancements and Line Item Management

Added bulk line item management with AI-powered column mapping, bulk targeting type and audience tabs, Campaign Deal IDs bulk-apply, Nexxen segment list API integration, and individual line item save functionality to the Campaign Requestor.

### Nexxen Integration Fixes and Data Factory Cleanup

Reverted unsupported `includeSegmentId` Nexxen API changes (Nexxen only supports `behavioralSegmentSetId` and `retargetingSegmentId`), fixed insertion-order DTO types, removed data factory sync mode configuration added in error, cleaned up ClickHouse destination defaults, and added a repair-statara-partitions job template.

### NexxenCredentialForwardFix

Stopped forwarding Nexxen credentials (`clientId`, `clientSecret`, `nexxenApiUrl`) from the data factory service to external-service, where they were being unnecessarily exposed.

### Revert "[HOTFIX] Salesforce Additional Fields in PCL"

Reverted the initial Salesforce Additional Fields in PCL change after issues were discovered post-merge; a corrected version was re-applied in a subsequent PR.

### Multiple Transactions Fix

Fixed the campaign transaction table showing duplicate entries by filtering out parent-campaign-level transactions, ensuring only the correct transaction records are displayed.

### Guard xAxis access when Google data is empty

Fixed a white-screen crash on the Paid Search (Google) module for accounts with no data by adding null and undefined guards around `xAxis` and `byDate` property accesses, and covered the empty-data paths with enzyme tests.

### Fix Paid Search Missing Dates

Fixed five compounding bugs that had frozen paid search reporting at 2026-05-07: stale unique-key conflicts preventing new date rows from saving, campaigns without an end date pulling 14 years of data and timing out, a missing null guard causing crashes on empty API responses, a chunk slice off-by-one dropping one record per thousand, and `concat(false)` polluting the consolidated date array. All fixes applied across both the Prisma v1 and Prisma 6 job paths.

### Stop ingestion scheduler infinite loop

Fixed two interacting bugs in `DataIngestionConsumer` that caused scheduled ingestion jobs to re-arm every few milliseconds instead of waiting for the configured interval, hammering external DSP APIs with production credentials. The safety-net now checks all active job states, and zero-delay values are clamped to a full sync-frequency interval so a cron-style schedule never fires immediately.

### Fix paid-search-prisma6 crashes: empty campaigns and Decimal aggregation

Fixed two production bugs in the paid-search-prisma6 job: campaigns that belonged to a different org's summary were no longer reconnected during update, causing a crash when dereferencing the result array; and Prisma 6's `Decimal` return type was being string-concatenated instead of numerically summed, producing malformed channel total values that failed Prisma's own decimal validation.
