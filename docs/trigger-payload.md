# ReleaseNotesAgent — API Trigger Payload

This document specifies the payload format for triggering `ReleaseNotesAgent` via the Paperclip Routines API.

---

## Trigger endpoint

```
POST /api/routines/{routineId}/trigger
Authorization: Bearer {PAPERCLIP_API_KEY}
Content-Type: application/json
```

---

## Payload schema

```jsonc
{
  // REQUIRED: The GitHub PR number of the deployment/release PR.
  // Used to extract the list of merged commits and linked tickets.
  "prNumber": 42,

  // REQUIRED: The semver version string for this release.
  // Used as the report filename (reports/releases/vX.Y.Z.md) and in the report header.
  "version": "v1.4.0",

  // OPTIONAL: Full GitHub PR URL. Preferred over prNumber when both are supplied
  // (allows cross-repo resolution without hardcoding the repo slug in the agent).
  "prUrl": "https://github.com/pacholoamit/med/pull/42",

  // OPTIONAL: Short description of the release (shown as the report subtitle).
  // If omitted, agent generates one from the PR title.
  "summary": "Adds the Relations pipeline and scheduled report agents.",

  // OPTIONAL: ISO-8601 deploy timestamp. Defaults to now() if omitted.
  "deployedAt": "2026-05-23T14:30:00Z",

  // OPTIONAL: Environment the release was deployed to (e.g. "production", "staging").
  // Shown in the report header. Defaults to "production".
  "environment": "production"
}
```

---

## Minimal valid payload

```json
{
  "prNumber": 42,
  "version": "v1.4.0"
}
```

---

## Full example payload

```json
{
  "prNumber": 42,
  "version": "v1.4.0",
  "prUrl": "https://github.com/pacholoamit/med/pull/42",
  "summary": "Adds the Relations pipeline and scheduled report agents.",
  "deployedAt": "2026-05-23T14:30:00Z",
  "environment": "production"
}
```

---

## Expected agent behaviour

1. Resolve the PR using `prUrl` (preferred) or `{GITHUB_REPO}/pulls/{prNumber}`.
2. Fetch all commits merged into the PR.
3. Parse commit messages and PR body for ticket references (`MED-\d+`, closes/fixes/resolves patterns).
4. For each ticket, fetch title and labels from the Paperclip API.
5. Group tickets by label (Feature, Bug, Chore, etc.).
6. Write `reports/releases/{version}.md` using the template below.
7. Commit and push to `med-relations` main branch.
8. GitHub Actions redeploys the Pages site automatically.

---

## Output report template

```markdown
# Release Notes — {version}

**Deployed:** {deployedAt}  
**Environment:** {environment}  
**PR:** [#{prNumber}]({prUrl})

{summary}

## What's new

### Features
- [MED-NN](/MED/issues/MED-NN) — Ticket title

### Bug fixes
- [MED-NN](/MED/issues/MED-NN) — Ticket title

### Chores / maintenance
- [MED-NN](/MED/issues/MED-NN) — Ticket title

---
*Generated by ReleaseNotesAgent*
```

---

## Error handling

| Case | Expected behaviour |
|---|---|
| PR not found | Fail the execution issue with a comment explaining the 404 |
| `version` already has a report | Overwrite with the new content and add `[updated]` to the commit message |
| No linked tickets found | Emit the report with a "No linked tickets found in this PR" note |
| GitHub API rate limit | Retry with exponential backoff (3 attempts), then fail with rate-limit details |
